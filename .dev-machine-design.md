## Architecture Summary

The provider implements the **Three-Medium Architecture**:

| Medium | Purpose | Target Size |
|--------|---------|-------------|
| **Python** | Mechanism — control flow, state machines, protocol translation | ~300 lines |
| **YAML** | Policy — error mappings, event routing, retry config | ~200 lines |
| **Markdown** | Contracts — behavioral requirements, invariants | ~400 lines |

**Core Pattern: Deny + Destroy**
- Every SDK session is ephemeral (create, use once, destroy)
- A `preToolUse` deny hook prevents SDK tool execution
- Amplifier's orchestrator handles all tool execution

**SDK Boundary: Contract-First Membrane**
- No SDK type crosses the adapter boundary
- Domain code never imports from SDK
- Three-layer: Adapter → Driver → Raw SDK

Full specification: [debates/GOLDEN_VISION_V2.md](debates/GOLDEN_VISION_V2.md)
